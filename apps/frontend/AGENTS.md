# AGENTS.md — ERAWADEE Catalog Cover

Этот файл задаёт правила для Codex/AI-агента при разработке обложки каталога продукции ERAWADEE. Цель — реализовать премиальную, чистую и адаптивную обложку каталога 2026 с визуальным языком тайской традиционной медицины, натуральной косметики и современного wellness-бренда.

## 1. Контекст проекта

Разрабатывается обложка каталога продукции ERAWADEE: традиционная медицина и косметика Таиланда. Обложка должна выглядеть премиально, спокойно и натурально: много воздуха, аккуратная сетка, зелёно-кремовая палитра, serif-заголовки и sans-serif основной текст.

Основные сообщения бренда:

- тайские традиции;
- натуральные ингредиенты;
- научные инновации;
- здоровье, красота, сила, мудрость, гармония, забота.

Если в проекте уже есть дизайн-система, компоненты, CSS-переменные или Tailwind-токены, сначала используй существующие решения и только затем добавляй недостающие токены.

## 2. Приоритеты при разработке

1. Не меняй стек проекта без необходимости.
2. Перед изменениями изучи структуру проекта: `package.json`, `src/`, `app/`, `pages/`, `components/`, `styles/`, `public/`.
3. Сохраняй существующий стиль кода, форматирование, способ импорта и архитектуру компонентов.
4. Все цвета, размеры и шрифты выноси в токены, CSS-переменные или theme-конфиг.
5. Не хардкодь брендовые цвета внутри компонентов, кроме случаев, где проект не имеет слоя стилей.
6. Визуальный результат важнее демонстрации технической сложности: интерфейс должен быть аккуратным, лёгким и премиальным.

## 3. Визуальное направление

Общее настроение: премиальный натуральный бренд, тайская традиция, чистота, доверие, мягкая медицина и косметика. Избегай агрессивных цветов, тяжёлых теней, перегруженных декоративных элементов и слишком современного «техно»-вида.

Визуальные принципы:

- много свободного пространства;
- строгая вертикальная структура сверху вниз;
- спокойные зелёные и кремовые фоны;
- тонкие разделители;
- акцентные лаймовые линии;
- декоративный тайский орнамент только как вторичный элемент;
- премиальные serif-заголовки;
- читаемый sans-serif для текста;
- мягкие тени, если нужен эффект печатной страницы или карточки.

Если есть расхождение между визуальным превью и текстовым ТЗ, приоритет такой:

1. утверждённая структура из ТЗ;
2. брендовая палитра и типографика;
3. визуальное превью как ориентир по настроению, воздуху и премиальности.

## 4. Структура страницы

Обложка состоит из 7 блоков сверху вниз.

### 4.1 Hero

Фон: тёмно-зелёный градиент.

Содержимое:

- бейдж `КАТАЛОГ ПРОДУКЦИИ 2026`;
- логотип ERAWADEE в круглом контейнере `80x80px`;
- название бренда `ERAWADEE`;
- подзаголовок `ТРАДИЦИОННАЯ МЕДИЦИНА И КОСМЕТИКА ТАИЛАНДА`;
- заголовок/описание каталога `Полный каталог натуральной продукции для здоровья, красоты и силы`;
- декоративный тайский бордюр слева, если доступен брендовый SVG/паттерн.

Требования:

- Hero должен быть главным визуальным акцентом.
- Логотип использовать как SVG с прозрачным фоном, если он есть в проекте.
- Не заменять логотип текстовым плейсхолдером, если доступен реальный asset.
- Если логотипа нет, использовать временный контейнер с понятной подписью, но оставить TODO-комментарий.

### 4.2 Accent line

Горизонтальная линия-разделитель высотой `4px`.

Фон: лаймовый градиент:

```css
linear-gradient(90deg, #8CB63C 0%, #A5D44B 50%, #8CB63C 100%)
```

### 4.3 About / О компании

Фон: кремовый `#F5F0E8`.

Содержимое:

- заголовок капсом `О КОМПАНИИ`;
- абзац о бренде;
- вертикальная лаймовая полоска-акцент слева.

Текст:

```text
ERAWADEE — премиальный бренд натуральной продукции, объединяющий вековые тайские традиции врачевания и достижения современной науки. Мы помогаем людям жить дольше и счастливее: каждый продукт создан на основе рецептов, передаваемых из поколения в поколение, с использованием только натуральных ингредиентов, прошедших контроль безопасности и эффективности. Наша продукция экспортируется в страны Европы и Азии и отмечена международными наградами за вклад в сохранение тайского культурного наследия.
```

### 4.4 Three pillars / Три столпа

Фон: белый.

Сетка: 3 колонки на desktop, 1 колонка на mobile.

Каждая колонка:

- SVG-иконка в зелёном круге;
- заголовок;
- краткий текст;
- тонкий разделитель между колонками на desktop.

Контент:

1. `ТАЙСКИЕ ТРАДИЦИИ`  
   `Рецепты, проверенные вековой историей Таиланда, в основе каждого продукта`

2. `НАТУРАЛЬНЫЕ ИНГРЕДИЕНТЫ`  
   `Только природные компоненты, проверенные на безопасность`

3. `НАУЧНЫЕ ИННОВАЦИИ`  
   `Современные технологии, раскрывающие весь потенциал природы`

Иконки:

- чаша с травами;
- листок;
- колба/пробирка.

Если SVG-иконок нет, использовать аккуратные временные outline-иконки и оставить TODO.

### 4.5 Values / Ценности

Фон: тёмно-зелёная полоса.

Теги:

```text
ЗДОРОВЬЕ · КРАСОТА · СИЛА · МУДРОСТЬ · ГАРМОНИЯ · ЗАБОТА
```

Требования:

- теги в одну строку на desktop;
- на mobile разрешён перенос строк;
- межбуквенный интервал повышенный;
- цвет текста — светлый лайм или кремовый, в зависимости от контраста.

### 4.6 Film CTA / Документальный фильм

Фон: кремовый `#F5F0E8`.

Содержимое:

- круглая Play-кнопка `56px`;
- заголовок `ДОКУМЕНТАЛЬНЫЙ ФИЛЬМ`;
- название `Лет тайской традиционной медицины`;
- описание `Полная история создания и развития системы тайского врачевания`;
- ссылка `youtube.com/watch?v=PLACEHOLDER`.

Поведение:

- клик по Play-кнопке и ссылке открывает YouTube в новой вкладке;
- использовать `target="_blank"` и `rel="noopener noreferrer"`;
- если реального URL нет, оставить плейсхолдер и TODO.

### 4.7 Footer

Фон: тёмно-зелёный.

Содержимое:

- слева: `ERAWADEE`;
- справа: `erawadee.com · hello@erawadee.com · +66 84 945 0447`.

На mobile контакты можно переносить в несколько строк.

## 5. Цветовая палитра

Используй следующие CSS-переменные как единый источник правды:

```css
:root {
  --brand-dark: #2D4A1E;
  --brand-olive: #3D6B2E;
  --brand-lime: #8CB63C;
  --brand-lime-light: #A5D44B;
  --cream: #F5F0E8;
  --gold: #C8A84E;
  --white: #FFFFFF;
  --text-dark: #2F332B;
  --text-muted: #6F7568;
  --border-soft: rgba(45, 74, 30, 0.14);
  --shadow-soft: 0 24px 60px rgba(45, 74, 30, 0.14);
}
```

Назначение цветов:

- `--brand-dark` — основной тёмно-зелёный, hero и footer;
- `--brand-olive` — градиенты и вторичные зелёные акценты;
- `--brand-lime` — иконки, кнопки, активные акценты;
- `--brand-lime-light` — подзаголовки, теги, лёгкие акценты;
- `--cream` — фон информационных блоков;
- `--gold` — зарезервирован для специальных декоративных элементов, не использовать массово.

## 6. Типографика

### Заголовки

Использовать serif-шрифт:

```css
font-family: "Cormorant Garamond", Georgia, "Times New Roman", serif;
```

Размеры:

- бренд `ERAWADEE`: `48–52px` на desktop, около `36px` на mobile;
- заголовок каталога: `26–28px`;
- заголовки секций: `15px`, uppercase, letter-spacing `0.18em–0.24em`.

### Основной текст

Использовать sans-serif:

```css
font-family: "Nunito Sans", Calibri, Arial, sans-serif;
```

Размеры:

- основной текст: `15px`;
- подписи и мелкие элементы: `11–12px`;
- веса: `300`, `400`, `600`.

Рекомендуемые настройки:

```css
body {
  font-family: "Nunito Sans", Calibri, Arial, sans-serif;
  color: var(--text-dark);
  line-height: 1.65;
}

h1,
h2,
h3,
.brand-title {
  font-family: "Cormorant Garamond", Georgia, "Times New Roman", serif;
  font-weight: 500;
  letter-spacing: 0.12em;
}
```

## 7. Layout и адаптивность

Desktop:

- основной контейнер: `max-width: 900px`;
- контейнер центрирован;
- блоки идут строго сверху вниз;
- три столпа расположены в 3 колонки;
- footer — бренд слева, контакты справа.

Mobile `<640px`:

- уменьшить горизонтальные отступы;
- три столпа перевести в 1 колонку;
- брендовый заголовок уменьшить до `36px`;
- логотип можно уменьшить до `64px`;
- CTA и footer перестроить в вертикальную компоновку;
- не допускать горизонтального скролла.

Рекомендуемые брейкпоинты:

```css
@media (max-width: 640px) {
  .catalog-cover {
    max-width: 100%;
  }

  .brand-title {
    font-size: 36px;
  }

  .pillars-grid {
    grid-template-columns: 1fr;
  }
}
```

## 9. CSS / стилизация

Предпочтительный порядок:

1. использовать существующую дизайн-систему проекта;
2. использовать CSS-переменные проекта;
3. если проект на Tailwind — добавить брендовые значения в theme или использовать CSS variables через arbitrary values;
4. если проект на CSS Modules — создать отдельный module-файл для компонента;
5. если проект на plain CSS — добавить отдельный файл стилей и импортировать его рядом с компонентом.

Запрещено:

- раскидывать цвета по компонентам без токенов;
- использовать случайные зелёные оттенки вне брендовой палитры;
- перегружать страницу анимациями;
- использовать тяжёлые градиенты, glassmorphism, neon-эффекты или яркие кислотные цвета;
- менять текст без необходимости.

Допустимо:

- мягкая тень у общего контейнера;
- тонкие бордеры;
- subtle hover для Play-кнопки;
- лёгкое увеличение/opacity на hover, если это не ломает премиальный стиль.

## 10. Assets

Ожидаемые assets:

- логотип ERAWADEE в SVG, прозрачный фон, слон Эраван;
- SVG-иконка чаши с травами;
- SVG-иконка листка;
- SVG-иконка колбы/пробирки;
- тайский орнамент для бордюров, опционально.

Правила:

- SVG хранить в `public/`, `assets/` или в существующей папке проекта для изображений;
- не вставлять большие SVG inline без необходимости;
- у изображений должен быть `alt`;
- декоративным изображениям можно ставить `aria-hidden="true"` и пустой `alt`, если они не несут смысла;
- не добавлять внешние URL для assets без необходимости.

## 11. Доступность

Обязательно:

- семантическая структура: `main`, `section`, `header`, `footer`;
- один главный `h1` на странице;
- корректная иерархия заголовков;
- достаточный контраст текста на тёмно-зелёном фоне;
- видимый focus-state для интерактивных элементов;
- Play-кнопка должна иметь понятный `aria-label`, например `Открыть документальный фильм на YouTube`;
- внешние ссылки должны использовать `rel="noopener noreferrer"`.

## 12. Проверка качества перед завершением

Перед финальным ответом Codex должен:

1. Запустить форматирование, если в проекте есть команда форматирования.
2. Запустить lint, если есть команда `lint`.
3. Запустить тесты, если они есть и не требуют внешней инфраструктуры.
4. Запустить build, если проект небольшой и команда доступна.
5. Проверить desktop и mobile layout.
6. Проверить, что нет горизонтального скролла на mobile.
7. Проверить, что все цвета берутся из токенов.
8. Проверить, что ссылки, alt и aria-атрибуты заданы корректно.

Типовые команды, если они есть в `package.json`:

```bash
npm run lint
npm run test
npm run build
```

Если проект использует `pnpm`, `yarn` или `bun`, используй тот пакетный менеджер, который уже зафиксирован lock-файлом.

## 13. Что не делать

- Не создавать новый дизайн вместо утверждённого ТЗ.
- Не менять палитру и типографику без причины.
- Не удалять существующие компоненты, если можно расширить их аккуратно.
- Не добавлять тяжёлые зависимости только ради этой обложки.
- Не использовать случайные иконки, если есть брендовые SVG.
- Не переводить русский текст на английский.
- Не заменять `ERAWADEE` другим написанием.
- Не публиковать placeholder YouTube как финальную ссылку без пометки TODO.

## 14. Definition of Done

Задача считается выполненной, если:

- реализованы все 7 блоков обложки;
- соблюдена палитра ERAWADEE;
- соблюдена типографика serif/sans-serif;
- desktop-версия центрирована и не шире `900px`;
- mobile-версия корректно перестраивается до одной колонки;
- контент совпадает с ТЗ;
- assets подключены или явно отмечены как TODO;
- интерактивные элементы доступны с клавиатуры;
- проект проходит доступные проверки качества.
